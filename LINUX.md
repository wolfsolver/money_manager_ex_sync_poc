# MMEX-Sync on Linux

This guide explains how to configure and run MMEX-Sync on a Linux system. Since the project uses Node.js Single Executable Application (SEA) and native modules like SQLite, specific steps are required.

## Prerequisites

1.  **Node.js**: It is recommended to have Node.js installed (version 20 or higher).
2.  **System Libraries**: Ensure your system has standard C libraries (glibc).

## Installing Linux Components

To run the application on Linux, three main components are required:

### 1. Main Executable (`mmex-sync-linux`)
This file is generated by the `build-linux.ps1` script on Windows.
- If you haven't already, run `.\build-linux.ps1` on Windows.
- Copy the `dist/mmex-sync-linux` file to your Linux system.
- Grant execution permissions:
  ```bash
  chmod +x mmex-sync-linux
  ```

### 2. Native SQLite Module (`better_sqlite3.node`)
Since SQLite is a native module, the Windows version will not work on Linux. You have two options:

**Option A (Recommended): Download prebuilt binary**
Download the file corresponding to your Node.js version from the official `better-sqlite3` releases:
- Go to: [WiseLibs/better-sqlite3 Releases](https://github.com/WiseLibs/better-sqlite3/releases/tag/v12.9.0)
- Expand the **Assets** section.
- Look for a file ending in `linux-x64.tar.gz` (e.g., `better-sqlite3-v12.9.0-node-v115-linux-x64.tar.gz` for Node 20).
- Extract the `better_sqlite3.node` file from the archive and place it in the same folder as the executable.

**Option B: Local compilation**
If you have `npm` and `python3/make/g++` on Linux:
```bash
npm install better-sqlite3
# Use the file generated in node_modules/better-sqlite3/build/Release/better_sqlite3.node
```

### 3. SQL Script (`tables_v1_for_sync.sql`)
Copy the `assets/sql/tables_v1_for_sync.sql` file to the same folder as the executable.

## Running the Application

Once you have all the files in the same folder:
```bash
./mmex-sync-linux
```

## Distribution Notes

For a "portable" distribution, the folder should contain:
- `mmex-sync-linux` (the executable)
- `better_sqlite3.node` (the database driver for Linux)
- `tables_v1_for_sync.sql` (the database schema)
